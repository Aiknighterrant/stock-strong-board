# 数据源说明

## 免费数据源列表

### 1. 东方财富网 (eastmoney.com)
**用途**：获取涨停股池和基础数据
**接口示例**：
- 涨停股列表：`http://quote.eastmoney.com/ztb/detail#type=ztgc`
- 股票详情：`http://quote.eastmoney.com/concept/sh600000.html`
**数据字段**：
- 股票代码、名称
- 最新价、涨跌幅
- 成交额、成交量
- 市盈率、市净率

### 2. 新浪财经 (sina.com.cn)
**用途**：获取封单金额和换手率数据
**接口示例**：
- 封单数据：`http://hq.sinajs.cn/list=sh600000`
- 实时行情：`http://hq.sinajs.cn/list=s_sh600000`
**数据字段**：
- 买一价、买一量（用于计算封单金额）
- 换手率
- 振幅
- 量比

### 3. 同花顺 (10jqka.com.cn)
**用途**：获取板块分类和表现数据
**接口示例**：
- 板块列表：`http://q.10jqka.com.cn/thshy/`
- 板块详情：`http://q.10jqka.com.cn/thshy/detail/code/881121/`
**数据字段**：
- 板块名称、代码
- 板块涨跌幅
- 板块成交额
- 领涨股

### 4. 腾讯财经 (qq.com)
**用途**：备用数据源，获取历史数据
**接口示例**：
- 日线数据：`http://qt.gtimg.cn/q=sh600000`
- 分时数据：`http://data.gtimg.cn/flashdata/hushen/minute/sh600000.js`

## 数据获取策略

### 1. 主备数据源
- 主要使用东方财富+新浪财经组合
- 同花顺作为板块数据主要来源
- 腾讯财经作为备用数据源

### 2. 数据验证
- 交叉验证关键数据（如成交额）
- 检查数据完整性（确保所有必需字段都有值）
- 数据一致性检查（如价格与涨跌幅匹配）

### 3. 错误处理
- 网络请求重试机制（最多3次）
- 数据源故障自动切换
- 部分数据缺失时的默认值处理

## 数据更新频率

### 实时数据（交易时间）
- 涨停股列表：每5分钟更新
- 封单数据：每1分钟更新
- 板块数据：每10分钟更新

### 历史数据
- 日线数据：每日收盘后更新
- 板块历史：每日收盘后更新

## 数据质量保证

### 1. 数据清洗
- 去除异常值（如价格为0或负值）
- 处理缺失值（使用前值填充或标记为NA）
- 数据格式标准化

### 2. 数据验证规则
- 成交额必须为正数
- 换手率在0-100%范围内
- 涨跌幅在±20%范围内（科创板±30%）
- 封单金额不超过流通市值

### 3. 数据缓存
- 短期缓存：5分钟（减少API调用）
- 长期缓存：1小时（非关键数据）
- 缓存失效策略：时间+事件驱动

## 注意事项

1. **数据延迟**：免费数据源通常有15-30分钟延迟
2. **数据限制**：部分接口有访问频率限制
3. **数据准确性**：免费数据源可能存在数据错误
4. **接口变更**：数据源接口可能不定期变更
