# 筛选规则详细说明

## 整体筛选流程

```
输入：交易日期
↓
步骤1：获取涨停股池（所有当日涨停股票）
↓
步骤2：封板强度筛选（成交额>1亿，封单金额异常值处理，封板成交比前25）
↓
步骤3：换手率筛选（10% ≤ 换手率 ≤ 20%）
↓
步骤4：板块强度筛选（板块涨跌幅>阈值，板块活跃度达标）
↓
输出：符合条件的强势封板股列表
```

## 步骤1：获取涨停股池

### 涨停定义
- **主板/中小板**：涨幅 ≥ 9.9%
- **创业板**：涨幅 ≥ 19.9%
- **科创板**：涨幅 ≥ 19.9%
- **ST股票**：涨幅 ≥ 4.9%

### 数据获取
1. 从东方财富获取当日所有涨停股票
2. 补充新浪财经的封单和换手率数据
3. 获取同花顺的板块分类数据
4. 数据清洗和验证

### 初始数据字段
```python
初始数据 = {
    "stock_code": "股票代码",
    "stock_name": "股票名称", 
    "close_price": "收盘价",
    "change_pct": "涨跌幅",
    "amount": "成交额(亿元)",
    "volume": "成交量(万手)",

    "sector_code": "板块代码",
    "sector_name": "板块名称"
}
```

## 步骤2：封板强度筛选

### 2.1 成交额筛选
**规则**：剔除当日成交额 ≤ 1亿元的股票
**理由**：成交额太小的股票流动性差，封板意义不大
**公式**：`amount > 1`（单位：亿元）

### 2.2 封单金额计算
**公式**：`board_amount = bid1_price × bid1_volume × 100 / 10000`
- `bid1_price`：买一价（元）
- `bid1_volume`：买一量（手，1手=100股）
- `× 100`：转换为股数
- `/ 10000`：转换为万元

### 2.3 异常值处理（IQR方法）
**目的**：排除封单金额异常高或异常低的极端值

**计算过程**：
1. 计算封单金额的四分位数
   - Q1 = 第25百分位数
   - Q3 = 第75百分位数
   - IQR = Q3 - Q1
2. 确定正常范围
   - 下限 = Q1 - 1.5 × IQR
   - 上限 = Q3 + 1.5 × IQR
3. 筛选规则
   - 保留：下限 ≤ board_amount ≤ 上限
   - 剔除：board_amount < 下限 或 board_amount > 上限

### 2.4 封板成交比计算
**公式**：`board_ratio = board_amount / (amount × 10000)`
- `board_amount`：封单金额（万元）
- `amount`：成交额（亿元）
- `× 10000`：亿元转换为万元（1亿元=10000万元）

**意义**：封板成交比越高，封板强度越大

### 2.5 前25名筛选
**规则**：按封板成交比降序排列，取前25名
**理由**：聚焦封板强度最高的股票
**注意**：如果股票数量不足25只，则取全部

## 步骤3：换手率筛选

### 3.1 默认阈值
- **下限**：10%
- **上限**：20%

### 3.2 筛选规则
**保留**：`10% ≤ turnover_rate ≤ 20%`
**剔除**：`turnover_rate < 10%` 或 `turnover_rate > 20%`

### 3.3 阈值调整逻辑
根据市场情况和股票特性动态调整：

#### 市场整体换手率水平
```python
if 市场平均换手率 < 5%:
    # 市场低迷，降低要求
    min_turnover = 8%
    max_turnover = 25%
elif 市场平均换手率 > 15%:
    # 市场活跃，提高要求
    min_turnover = 12%
    max_turnover = 18%
```

#### 股票市值大小
```python
if 市值 > 500亿:  # 大盘股
    min_turnover = 8%   # 降低下限
    max_turnover = 15%  # 降低上限
elif 市值 < 50亿:  # 小盘股
    min_turnover = 15%  # 提高下限
    max_turnover = 25%  # 提高上限
```

#### 新股处理
```python
if 上市天数 < 20:  # 新股
    min_turnover = 20%  # 提高下限
    max_turnover = 40%  # 提高上限
```

## 步骤4：板块强度筛选

### 4.1 板块涨跌幅筛选
**默认阈值**：`sector_change > -0.5%`
**调整规则**：
- 牛市环境：`sector_change > 0%`
- 熊市环境：`sector_change > -1%`
- 震荡市：`sector_change > -0.5%`

### 4.2 板块活跃度筛选
**指标1**：板块成交额占比
- 计算：`sector_amount_ratio = sector_amount / total_market_amount`
- 要求：`sector_amount_ratio > sector_avg_ratio × 0.8`

**指标2**：板块内涨停股数量
- 要求：`sector_limit_up_count ≥ 2`（小板块）或 `≥ 5`（大板块）

**指标3**：板块资金流向
- 要求：`sector_net_inflow > 0`

### 4.3 综合板块强度评分
使用板块强度评分算法（详见sector_classification.md）
**要求**：`sector_score ≥ 60`

## 参数调整指南

### 核心参数说明

| 参数名 | 默认值 | 调整范围 | 影响 |
|--------|--------|----------|------|
| min_amount | 1亿元 | 0.5-5亿元 | 成交额门槛，值越大筛选越严格 |
| min_turnover | 10% | 5-15% | 换手率下限，值越大要求越活跃 |
| max_turnover | 20% | 15-30% | 换手率上限，值越小排除投机股 |
| sector_threshold | -0.5% | -2%到1% | 板块涨跌幅要求，值越高要求越强 |
| top_n | 25 | 10-50 | 封板成交比排名数量 |
| sector_score_min | 60 | 40-80 | 板块强度最低分 |

## 特殊股票处理

### 1. ST股票
**处理规则**：
- 单独标记，不参与普通股票排名
- 换手率要求：5-15%
- 封板成交比要求：>1.0
- 需要特别风险提示

### 2. 新股（上市<20天）
**处理规则**：
- 换手率要求：20-40%
- 封板成交比要求：>0.8
- 不参与板块强度筛选（板块数据不足）

### 3. 大盘股（市值>1000亿）
**处理规则**：
- 换手率要求：5-15%
- 封单金额要求：>5000万元
- 板块要求可适当放宽

### 4. 科创板/创业板
**处理规则**：
- 涨跌幅阈值：19.9%
- 换手率要求：15-30%
- 封板成交比要求：>0.6

### 综合筛选得分计算
```
综合筛选得分 = 
  封板成交比得分 × 0.4 +
  换手率得分 × 0.2 +
  板块强度得分 × 0.3 +
  成交额得分 × 0.1
```

### 结果排序
1. 按综合筛选得分降序排列
2. 得分相同时，按封板成交比降序排列
3. 再次相同时，按成交额降序排列
