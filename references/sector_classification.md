# 板块分类体系

## 主要板块分类标准

### 1. 申万行业分类 (申万宏源)
**特点**：最常用的行业分类标准，分为三级
**一级行业**：31个，如银行、非银金融、医药生物等
**二级行业**：134个，如银行Ⅱ、证券Ⅱ、化学制药等
**三级行业**：346个，如国有大型银行Ⅱ、证券Ⅲ、化学原料药等

### 2. 中信行业分类
**特点**：中信证券制定，分为三级
**一级行业**：30个，如银行、非银行金融、医药等
**二级行业**：109个，如银行、证券、化学制药等
**三级行业**：287个，如国有银行、证券、原料药等

### 3. 同花顺行业分类
**特点**：同花顺自建分类，实时更新
**一级行业**：66个，如银行、证券、化学制药等
**特点**：更贴近市场热点，包含新兴行业

### 4. 概念板块分类
**特点**：按市场热点概念划分
**常见概念**：
- 人工智能、芯片、5G、新能源车
- 光伏、储能、氢能源、锂电池
- 元宇宙、数字经济、信创、数据要素
- 医药医疗、消费电子、白酒、猪肉

## 本技能使用的板块体系

### 主要使用：同花顺行业分类
**原因**：
1. 数据获取方便（同花顺提供免费接口）
2. 更新及时（实时反映市场变化）
3. 覆盖全面（包含传统行业和新兴概念）

### 板块强度评估指标

#### 1. 板块涨跌幅
- **计算**：板块内所有成分股涨跌幅的加权平均
- **权重**：按流通市值加权
- **阈值**：板块当日涨跌幅 > -0.5%（默认）

#### 2. 板块成交额占比
- **计算**：板块成交额 / 全市场成交额
- **意义**：反映板块资金关注度
- **阈值**：板块成交额占比 > 板块平均占比的80%

#### 3. 板块内涨停股数量
- **计算**：板块内涨停股票数量
- **意义**：反映板块赚钱效应
- **阈值**：涨停股数量 ≥ 2（小板块）或 ≥ 5（大板块）

#### 4. 板块资金流向
- **计算**：板块资金净流入额
- **数据源**：同花顺资金流向数据
- **阈值**：资金净流入 > 0

## 板块强度评分算法

### 综合评分公式
```
板块强度得分 = 
  涨跌幅得分 × 0.4 +
  成交额占比得分 × 0.3 +
  涨停股数量得分 × 0.2 +
  资金流向得分 × 0.1
```

### 各项得分计算
#### 1. 涨跌幅得分（0-100分）
```python
if change_pct >= 5%: score = 100
elif change_pct >= 3%: score = 80 + (change_pct-3%)×10
elif change_pct >= 0%: score = 60 + change_pct×6.67
elif change_pct >= -1%: score = 40 + (change_pct+1%)×20
else: score = max(0, 40 + change_pct×40)
```

#### 2. 成交额占比得分（0-100分）
```python
# 与板块历史平均占比比较
ratio = current_ratio / historical_avg_ratio
if ratio >= 1.5: score = 100
elif ratio >= 1.2: score = 80 + (ratio-1.2)×66.67
elif ratio >= 1.0: score = 60 + (ratio-1.0)×100
elif ratio >= 0.8: score = 40 + (ratio-0.8)×100
else: score = ratio × 50
```

#### 3. 涨停股数量得分（0-100分）
```python
# 根据板块大小调整
if sector_size == "large":  # 大板块（成分股>50）
    if limit_up_count >= 10: score = 100
    elif limit_up_count >= 5: score = 60 + (limit_up_count-5)×8
    else: score = limit_up_count × 12
else:  # 小板块（成分股≤50）
    if limit_up_count >= 5: score = 100
    elif limit_up_count >= 3: score = 60 + (limit_up_count-3)×20
    else: score = limit_up_count × 20
```

#### 4. 资金流向得分（0-100分）
```python
if net_inflow > 1e8:  # 净流入>1亿
    score = 100
elif net_inflow > 0:
    score = 60 + (net_inflow/1e8)×40
elif net_inflow > -5e7:  # 净流出<5000万
    score = 40 + (net_inflow+5e7)/5e7×20
else:
    score = max(0, 40 + net_inflow/1e8×80)
```

## 板块筛选规则

### 1. 必过条件
- 板块当日涨跌幅 > -0.5%
- 板块成交额占比 > 板块平均占比的50%
- 板块强度得分 ≥ 60分

### 2. 优选条件（满足任意2条）
- 板块涨跌幅 > 1%
- 板块成交额占比 > 板块平均占比
- 涨停股数量 ≥ 3
- 资金净流入 > 0

### 3. 排除条件（满足任意1条即排除）
- 板块涨跌幅 < -2%
- 板块成交额占比 < 板块平均占比的30%
- 板块内跌停股数量 ≥ 3
- 板块强度得分 < 40分

## 特殊板块处理

### 1. 银行、非银金融板块
- 降低涨跌幅权重（0.2→0.1）
- 提高成交额权重（0.3→0.4）
- 涨停股数量阈值降低（5→2）

### 2. ST板块
- 单独处理，不参与普通板块评估
- 需要特别风险提示

### 3. 新股板块
- 上市不满20个交易日的股票
- 换手率阈值调整（10-20% → 20-40%）
- 封板成交比要求更高（>0.8）

## 数据更新与维护

### 1. 板块成分股更新
- 每月检查一次板块成分股变化
- 新股上市后及时归类
- ST股票标记和调整

### 2. 板块权重调整
- 每季度重新计算板块平均成交额占比
- 根据市场变化调整评分权重
- 更新板块大小分类（大/小板块）

### 3. 新兴概念识别
- 监控市场新出现的概念板块
- 用户反馈收集和概念验证
- 新概念板块的测试和纳入
